# geo-insar 报告解读指南

> 这份文档帮你**读懂** InSAR 报告里的每一张图、每一个数字、每一个标注的点位。
> 适用范围:本平台 geo-insar 出的标准产物,以及 geo-reporter 自动生成报告里的 InSAR 章节,以及行业里常见的第三方 InSAR 报告(LiCSAR、商业 PS-InSAR 服务等)。

---

## 0. 一份典型 InSAR 报告长什么样

完整的 InSAR 报告(无论是矿区监测、城市沉降、滑坡评估)通常由四部分组成:

```
┌────────────────────────────────────────────────────────────┐
│                                                            │
│  ① 数据概览(Data Overview)                                │
│     · AOI 范围、卫星、时间窗、干涉对数量、整体相干性         │
│                                                            │
│  ② 空间产品(Spatial Products)                            │
│     · 速率图(mm/yr 双向 colormap)                        │
│     · 累计形变图(cumulative displacement)                 │
│     · 相干性图(0-1)                                       │
│     · 异常分布图(超阈值像素聚类)                          │
│                                                            │
│  ③ 时序产品(Temporal Products)                           │
│     · 关键点位 P1, P2, P3, ...(时间序列曲线)              │
│     · 季节性 / 趋势分解                                    │
│     · 加速 / 减速段落                                      │
│                                                            │
│  ④ 解读与建议(Interpretation)                            │
│     · 量级是否合理                                          │
│     · 与地质/开采活动的关联                                 │
│     · 监测建议                                              │
│                                                            │
└────────────────────────────────────────────────────────────┘
```

**本平台当前产出 ①②**,**③ 时序分析在 geo-analyser 子系统**(insar_timeseries.py 简化版 SBAS),**④ 解读由 geo-reporter 自动生成**(基于 LLM + 本地统计)。

## 1. 数据概览栏的关键字段

报告开头通常有一张参数表,**你看到这些数字就要本能反应"意味着什么"**:

| 字段 | 含义 | 怎么读 |
|---|---|---|
| **AOI 面积** | km² | < 600 是 burst 模式;> 2500 是整景。决定后端选择 |
| **时间跨度** | YYYY-MM-DD 到 YYYY-MM-DD | 长 = 能看慢变信号;短 = 只能看快变(地震/塌陷) |
| **干涉对数量** | N | < 5 几乎只能定性;30+ 才能做时序;100+ 可做 PS/SBAS |
| **平均时间基线** | 天 | 12-24 天 = 高相干;> 60 天 = 警惕去相干 |
| **整体相干性均值** | 0-1 | > 0.5 优秀;0.3-0.5 可用;< 0.3 不可信 |
| **轨道方向** | Asc / Desc | 单轨 → LOS 形变;升降轨联合 → E/N/U 三分量 |
| **极化** | VV / HH | Sentinel-1 几乎都是 VV |
| **数据来源** | hyp3_gamma / hyp3_isce_burst / snap_local / licsar | 决定处理算法链 |

**红旗信号**:
- 相干性均值 < 0.3 → 报告结论可信度低,**重新选时间窗或换季节**
- 平均时间基线 > 60 天 → 农田/植被区基本没用,**改用 ALOS-2 L 波段**
- 干涉对 < 5 → 不要做时序结论,**只能定性**

## 2. 空间产品图怎么读

### 2.1 速率图(velocity, mm/year)

**最常见的封面图**。RdBu_r 双向 colormap:

```
红色  ←─── 远离卫星(沉降) ─── 0 ─── 靠近卫星(抬升) ───→  蓝色
-30 mm/yr                                                +30 mm/yr
```

**关键读图技巧**:
- **看色值范围**(色条 vmin/vmax):本平台默认对称截断(±max(|p2|, |p98|)),让 0 永远在 colormap 中心,**不要被自动拉伸误导**
- **看连续区块 vs 噪声**:真实形变是**空间连续**的色块(几公里到几十公里),零星红点蓝点多半是相干噪声
- **看相干性掩膜**:本平台默认 coherence < 0.3 的像素设为 NaN(透明),颜色出现的地方都是可信像素
- **参考点位置**:速率图是相对的,所有点都参照某个"零形变假设点"。报告应该标注参考点(通常在 AOI 外的稳定区)

### 2.2 累计形变图(cumulative displacement, mm)

某个 acquisition 相对初始日期的总形变量。**和速率图的区别**:
- 速率 = 斜率(线性回归出来的)
- 累计 = 真实测量(某时刻的位移量)

如果地表是匀速沉降:累计形变 = 速率 × 时间。**非匀速**(季节性、突发)就只有累计形变图能看出来。

### 2.3 相干性图(coherence, 0-1)

**质量地图**。每个像素一个值:
- 黑/深红 (< 0.2):噪声区(水、密植被、突变形变)
- 黄/绿 (0.3-0.5):边缘可用
- 白/亮黄 (> 0.6):高质量

**怎么用**:
- 在解读速率图之前,**先看相干性图**。低相干区的速率值不要当真
- 矿区中心相干性突然降低可能意味着剧烈形变(相位卷绕超过了 28 mm)
- 季节性变化:夏季农田 coherence ↓,冬季城市 coherence ↑

### 2.4 异常分布图(active region mask)

**经常以"激活区/沉降区"形式呈现**。0/1 二值图,标注"超过阈值的活跃像素 + 连通块过滤"后的结果。

geo-analyser 的 `los_velocity_clustering()` 输出就是这种图。**用 DBSCAN 思路**(连通块标记 + 最小尺寸过滤),把零星噪声去掉。

## 3. 时序产品 — 关键点位 P1, P2, P3 是怎么选出来的(重点)

时序 InSAR 的核心成果是**几个代表性点位的时间序列曲线**,通常报告里标注为 P1, P2, P3...,叠加在速率图上。

**这些点不是随便选的**,有明确的算法逻辑。下面分两种主流时序方法分别讲。

### 3.1 PS-InSAR(Permanent Scatterer)的点位选定

**核心思想**:不是所有像素都适合做时序——只有"反射特性稳定的散射体"(persistent scatterer)才能给出可信的相位时间序列。

#### 第一步:候选点筛选(用振幅离差指数 ADI)

对每个像素,在 N 幅 SLC 影像上的**振幅时间序列** A = [A_1, A_2, ..., A_N],计算:

```
ADI = σ_A / μ_A     (振幅标准差 / 振幅均值)
```

- **物理含义**:振幅在多次拍摄中**变化越小**,该点散射特性越稳定
- **阈值**:`ADI < 0.25`(严格)或 `ADI < 0.40`(宽松)
- **代表点类型**:
  - 建筑物角 / 桥梁 / 大型岩石(ADI ≈ 0.10-0.20,极优秀)
  - 裸岩 / 矿区机械设备(ADI ≈ 0.20-0.30)
  - 农田 / 林地(ADI > 0.40,**不会被选为 PS**)

**典型矿区报告**:N=50 个 acquisition 的 AOI,通常能筛出 10⁴-10⁵ 个 PS 候选点。

#### 第二步:相位标准差二次筛选(StaMPS 方法)

ADI 只看振幅,还可能漏选"振幅稳但相位有大气延迟"的点。所以二次筛选:

1. 把 ADI 候选点的**残余相位**(扣除已知形变模型后)做时间标准差
2. 保留 `σ_phase < 1 rad`(约 ±4.5 mm)的点
3. 这一步通常能再筛掉 30%-50%

最终保留下来的就是**真正的 PS 点**。

#### 第三步:报告里展示哪些 P1, P2, P3?

报告作者从 N₂ 个 PS 点中**人工或自动挑出几个代表性点**,通常按这几个标准:

| 选定原则 | 含义 | 典型用法 |
|---|---|---|
| **最大形变点** | 时间序列绝对值最大 | "P1 - 最大沉降中心" |
| **最快加速点** | 二阶导数最大,形变速率变化剧烈 | "P2 - 异常加速区" |
| **代表性区域中心** | 用空间聚类找异常区,在每个簇中心取 1 个 | "P3 - 南矿区中心" |
| **稳定参考点** | 形变 ≈ 0,作为相对参考 | "P_ref - 参考点" |
| **已知工程位置** | 矿口、坝顶、监测桩、塔基等 | "P4 - 主井口" |
| **空间均匀分布** | 在 AOI 四角 + 中心各取 1 个 | 看空间一致性 |

**一份 8 页的矿区监测报告**通常展示 **5-10 个 P 点**,带每个点的时间序列曲线 + 速率拟合 + 95% 置信区间。

### 3.2 SBAS(Small Baseline Subset)的点位选定

**与 PS 的区别**:PS 适合"点状散射体"(城市、矿区机械),SBAS 适合"分布式散射体"(裸地、岩石面)。

#### 候选点筛选(用相干性矩阵)

对每个像素,看它在所有 N(N-1)/2 个干涉对里的相干性 [γ_{12}, γ_{13}, ..., γ_{N-1,N}]:

1. 选短基线对(`max_temporal_baseline < 阈值`,默认 36 天)→ 得到子集
2. 在这个子集里,**多对相干性都高**(中位数 > 0.7)的像素 = 候选点
3. 阈值不同方法不同:
   - 经典 SBAS:多对中位数 γ > 0.6
   - SqueeSAR(改进版):用相位三角形闭合误差 < 0.3 rad

**本平台 `insar_timeseries.temporal_velocity_trend()`** 是简化版 SBAS:
```python
# 简化算法(geo-analyser/insar_timeseries.py)
for each 对 (date1, date2, disp):
    rate_per_pair = disp * (365.25 / dt_days)   # mm/year
# 像素级取所有对的中位数 → 鲁棒于个别对的噪声
median_rate = nanmedian(rates_3d, axis=0)
```

这个简化版**不做点位筛选**,而是给出**全图速率场**。报告中的 P1/P2/P3 由后处理在速率场上挑出来。

#### P1, P2, P3 选定(SBAS 版)

跟 PS 类似,但因为输入是密集速率场(不是离散点),选法是:

1. **从异常分布图(§2.4)拿到活跃区掩膜**(连通块过滤后的高速率区)
2. 每个连通块取一个代表点:
   - **块中心**(几何中心或质心)
   - 或**块内速率最大点**
3. 加上稳定参考点(coherence > 0.7、|velocity| < 1 mm/yr)

**报告里的标注规则**:
- P1, P2, P3...:**活跃形变点**(按速率绝对值降序)
- P_ref 或 R:**参考点**(稳定区,所有形变值都相对它)
- P_k:**已知工程位置**(用户在地图上手动标注)

### 3.3 时间序列曲线怎么读

每个 P 点都会画一张时间序列图:

```
形变 (mm)
    │
  10│                    ●
    │                  ●     ● 95% CI(误差棒)
   0├──●─────●─────● ── ─── ─ ──   ← 参考日期(初始 0)
    │           ●
 -10│                          ●
    │                              ●
 -20│   线性拟合 v = -5 mm/yr           ●
    │
    └─────────────────────────────────────→ 时间
       2024-01   2024-04   2024-07   2024-10
```

**怎么读**:
- **斜率** = 速率(mm/year),最重要的数字
- **抖动幅度** vs **置信区间**:抖动比 CI 大很多 → 真实非线性形变;一样大 → 全是噪声
- **断点**:某时刻速率突然变 → 矿区开采状态变化 / 触发事件
- **季节性**:正弦波形 → 地下水位 / 冻融循环(矿区典型 ±5-10 mm)
- **离群点**:某一两个点偏离很远 → 大气延迟或解缠错误,**不要当真**

### 3.4 量纲与符号约定(报告常踩坑)

| 量 | 单位 | 正负含义 |
|---|---|---|
| LOS displacement | mm | **负 = 远离卫星(沉降)**;正 = 抬升 |
| LOS velocity | mm/year | 同上 |
| 累计沉降 | mm 或 cm,**绝对值** | 通常用"沉降量"代替"位移量"避免符号歧义 |
| 时间基线 | 天 | 总是正(slave - master) |
| 垂直基线 B⊥ | 米 | 一般给绝对值 |

**避免误读**:报告里如果说"P1 沉降 50 mm",量级是 50;如果说"P1 LOS 位移 -50 mm",量级也是 50(负号代表方向)。**两种说法在工程上等价,在物理上不同**。

## 4. 可信度与误差怎么读

### 4.1 不要被报告里的数字唬住

InSAR 报告经常写"P1 沉降速率 -8.34 ± 0.42 mm/year"——这种**两位小数 + 误差棒**的写法,**误差范围才是真信号**:

| 标称速率 | 误差棒 | 怎么解读 |
|---|---|---|
| -8.3 ± 0.4 mm/yr | < 5% 标称值 | 高质量,可作工程依据 |
| -8.3 ± 2.0 mm/yr | ~25% | 趋势可信,精确值参考 |
| -8.3 ± 5.0 mm/yr | > 60% | **趋势都不一定可靠**,只能定性 |

误差大主要来源:
- 大气延迟(几 cm)
- 解缠错误(整数倍 28 mm 跳变)
- 时间序列长度短(< 30 个点)
- 参考点不稳

### 4.2 InSAR 形变与 GPS / 水准测量对比

| 方法 | 量程 | 精度 | 空间密度 | 时间频率 |
|---|---|---|---|---|
| InSAR(C 波段) | 单对 ~10 mm,时序 ~ mm/yr | ±2-5 mm/yr | 数百万点/AOI | 6-12 天 |
| GPS 连续站 | 任意 | ±1 mm/yr | 1 个点 | 秒级 |
| 水准测量 | ~10 km 网内 | ±0.5 mm | 几十-几百点 | 半年-年 |

**InSAR 优势**:**密度**(每个像素 = 一个监测点)+ **回看历史**(2014 年起的 Sentinel-1 档案)
**InSAR 劣势**:**只能测 LOS**(2D 投影),GPS 测 3D;需要校准到 GPS 才能给出绝对值

## 5. 常见的"误读"陷阱

| 陷阱 | 实际情况 |
|---|---|
| "整片红色 = 整片在沉降" | 检查是否做了**参考点归零**:如果参考点本身在抬升,整片都被减成红色 |
| "0 形变 = 地表稳定" | 可能只是**相干性差,数据不可信**;先看相干性图 |
| "矿区中心没信号" | 形变太快,**相位卷绕了**;改短时间基线或换 L 波段 |
| "单对差异 30 mm 就是真形变" | C 波段 2π = 28 mm,**可能多了/少了一圈解缠**;看相邻对是否一致 |
| "速率图越红越严重" | 颜色被自动拉伸:**先看色条范围** |
| "时间序列里 1 月有突跳" | 多半是**冬季积雪导致相干性下降**,不是真形变 |
| "P1 显示加速沉降" | 看误差棒,可能 P1 在低相干区,加速是噪声 |

geo-reporter 已经接入了 InSAR 章节(第 8 章 "InSAR 形变监测资料"),自动从 geo-insar 标准输出读统计数据(`data_sources.fetch_insar_local()`),与 Tavily 网络检索的外部文献融合。

### 6.1 当前能自动出的内容

| 项 | 当前 | 来源 |
|---|---|---|
| 干涉对数量 | ✅ | metadata.json |
| 时间跨度 | ✅ | metadata.json |
| 整体相干性均值 | ✅ | stats.coherence_mean |
| 时空基线分布 | ✅ | metadata.json |

### 6.2 当前**还没**自动出的(后续 Phase 3.x)

- ❌ 速率图渲染(只有数值,没有 PNG 嵌入报告)

- ❌ 季节性 vs 趋势分解
- ❌ 与 GPS / 水准的交叉对比

这些需要 PS-InSAR 或 SBAS 完整时序处理,目前只在 `geo-analyser/insar_timeseries.py` 提供了**简化版速率反演**(没有 P 点选定)。如果业务需要正式报告里有 P 点和时序曲线,需要做 Phase 3.x 升级:
- 集成 MintPy(开源 SBAS/PS 工具链)
- 或对接商业 PS-InSAR 服务

## 7. 一个完整的示例报告解读(假设)

假设我们对庙山金矿跑完一个 6 个月时序,报告大概会有这些内容,**每段后给出怎么读**:

> **"AOI:zhaoyuan_miaoshan,面积 5.96 km²;36 个干涉对,时间跨度 6 个月。"**
**读法**:对数 > 30,够做时序;面积小,burst 模式合适。

> **"整体相干性中位数 0.42。"**
**读法**:中等。城市/矿区部分应该 > 0.5,植被覆盖区可能 < 0.3,看相干性图判断。

> **"速率图(图 1):AOI 中心(120.44°E, 37.19°N)出现 ~30 mm/yr 沉降漏斗,半径约 200 m。"**
**读法**:量级合理(深井金矿 cm/yr 级),空间尺度合理(采空区数百米)。**值得追踪**。

> **"P1 位于沉降中心(120.4385°E, 37.1925°N),速率 -32.6 ± 1.8 mm/yr,时间序列呈线性。"**
**读法**:误差棒 < 6%,高可信。线性 = 稳定塌陷,无加速。

> **"P2 位于沉降漏斗北缘(120.4421°E, 37.1948°N),速率 -12.3 ± 3.5 mm/yr,时间序列有季节波动。"**
**读法**:边缘点正常衰减(中心快、边缘慢)。季节波动可能是地下水位变化,**不影响沉降趋势**。

> **"P3 为参考点(距沉降漏斗 1.5 km,coherence 0.71),速率 +0.4 ± 0.6 mm/yr。"**
**读法**:✅ 参考点稳定(|v| < 误差棒)。**没有这个参考点,前面所有数字都是"相对值"没意义**。

## 8. 操作小抄:从产物到报告的关键命令

### 8.1 在 QGIS 里打开 InSAR 产物

```
1. 添加栅格图层 → los_displacement.tif
2. Layer Properties → Symbology:
   - 选 "Single band pseudocolor"

   - Min/Max 用 "Cumulative count cut"(2% - 98%)
3. 叠加 coherence.tif 作为掩膜(用 Identify by data 工具)
```

## 9. 总结:看 InSAR 报告的 3 个铁律

1. **先看相干性图,再看速率图** — 没有质量保证的速率值都是噪声
2. **先找参考点,再看绝对值** — 没有参考点的"沉降量"都是相对值
3. **先看误差棒,再看小数点** — 报告里写 -8.34 mm/yr,误差棒 ±2.0 时,真实值是"约 -8 mm/yr"

记住这三条,InSAR 报告里 90% 的坑都能避开。
